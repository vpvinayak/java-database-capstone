# Smart Clinic User Stories

## Doctor User Stories

### Story 1
**As a doctor, I can log in securely so that I can access my dashboard.**  
Priority: High  
Story Points: 3  
Acceptance Criteria:
- Login requires valid credentials
- Unauthorized users are denied access
- Dashboard loads after login

### Story 2
**As a doctor, I can view patient appointments so that I can manage consultations.**  
Priority: High  
Story Points: 5  
Acceptance Criteria:
- Appointments display by date
- Doctor can filter appointments
- Appointment details are visible

### Story 3
**As a doctor, I can access patient medical records so that I can prescribe accurate treatment.**  
Priority: High  
Story Points: 5  
Acceptance Criteria:
- Patient history is visible
- Records are read-only for doctors
- Prescription section is available

## Patient User Stories

### Story 1
**As a patient, I can register and log in so that I can access clinic services.**  
Priority: High  
Story Points: 3  
Acceptance Criteria:
- Registration validates email
- Login requires credentials
- User profile is created

### Story 2
**As a patient, I can search doctors by specialty and availability so that I can book appointments.**  
Priority: High  
Story Points: 5  
Acceptance Criteria:
- Search supports specialty
- Available time slots display
- Matching doctors appear

### Story 3
**As a patient, I can view my appointments so that I can track upcoming visits.**  
Priority: Medium  
Story Points: 3  
Acceptance Criteria:
- Upcoming appointments show
- Past appointments are separated
- Appointment status is visible

## Admin User Stories

### Story 1
**As an admin, I can add doctors so that clinic data stays updated.**  
Priority: High  
Story Points: 5  
Acceptance Criteria:
- Doctor details can be entered
- Validation prevents empty fields
- Doctor appears after saving

### Story 2
**As an admin, I can edit doctor profiles so that information remains accurate.**  
Priority: High  
Story Points: 3  
Acceptance Criteria:
- Existing doctor data loads
- Updates save correctly
- Changes reflect immediately

### Story 3
**As an admin, I can delete doctor profiles so that inactive doctors are removed.**  
Priority: Medium  
Story Points: 3  
Acceptance Criteria:
- Confirmation required
- Deleted doctor removed from listings
- Related records remain safe

### Story 4
**As an admin, I can manage appointments so that scheduling conflicts are minimized.**  
Priority: High  
Story Points: 5  
Acceptance Criteria:
- Appointment list is visible
- Admin can cancel appointments
- Schedule conflicts are prevented

### Story 5
**As an admin, I can monitor system records so that clinic operations stay organized.**  
Priority: Medium  
Story Points: 3  
Acceptance Criteria:
- Reports are accessible
- User activity is visible
- System records load correctly
